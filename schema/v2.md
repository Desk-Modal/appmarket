# AppMarket Schema — v2

The canonical machine-readable catalog that every DeskModal session
queries on startup (or marketplace refresh) to learn about every
published plugin, service, script, and app.

## Served from

```
https://raw.githubusercontent.com/Desk-Modal/appmarket/main/index.json
```

This URL is backed by the Fastly-fronted GitHub raw CDN: free, globally
distributed, aggressively cached, unlimited reads, no authentication.
The DeskModal runtime does **one** fetch of this file and has everything
it needs to populate the marketplace, resolve installs, verify
signatures, check dependencies, and stream icons.

Every published version of this file is also signed with the DeskModal
Ed25519 publisher key and the detached signature is served at
`index.json.sig` next to the JSON.

## `PluginEntry`

Every entry describes one installable unit — a plugin app, a service,
or a script. The runtime picks the platform asset whose key matches its
host, falling back to `wasm` if no native match exists, and refusing to
install if nothing matches and `wasm` is absent.

## Rules the aggregator enforces

1. **`id` must be globally unique.** The aggregator refuses to write if
   two entries collide.
2. **`latest_version`** is always SemVer; the aggregator picks the
   largest version tag that is not a pre-release, not a draft, and not
   marked `featured=false, deprecated=true`.
3. **Every entry must have at least one usable platform**. Either a
   native tarball matching the current platform matrix, or a `wasm`
   entry. Entries with neither are dropped with a warning in the build
   log.
4. **`sha256` is computed by the aggregator**, never copied blindly
   from the upstream `checksums.txt`. If the aggregator downloads the
   asset and recomputes to a different hash, the entry is dropped with
   an error — the release is considered tampered.
5. **`signature.checksums_url`** must resolve to a file that verifies
   against `publisher_key_id`. If it doesn't, the entry is dropped.
6. **`min_deskmodal_version`** comes from `plugin.toml`'s
   `[compat] min_deskmodal`. Missing value defaults to `"0.0.0"` with
   a warning.
7. **Dependencies** are resolved lazily at install time, not by the
   aggregator. The aggregator only validates that the `id` string
   parses.

## Client consumption

```rust
// pseudocode
let index: PluginIndex = fetch_with_etag_cache(
    "https://raw.githubusercontent.com/Desk-Modal/appmarket/main/index.json",
).await?;

// Verify the index itself before trusting it
let sig = fetch_bytes(
    "https://raw.githubusercontent.com/Desk-Modal/appmarket/main/index.json.sig",
).await?;
verify_ed25519(&index.raw_bytes(), &sig, &TRUSTED_KEYS["deskmodal-primary"])?;

// Pick a plugin, resolve host platform
let entry = index.catalog.iter().find(|e| e.id == "desk-modal.paper-trading")?;
let host = PlatformId::current();  // e.g. Win32X64
let asset = entry.platforms.get(&host)
    .or_else(|| entry.platforms.get(&PlatformId::Wasm))
    .ok_or(NoCompatiblePlatform)?;

// No extra API calls. sha256 + url + sig pointers are already in the index.
download_and_verify(asset).await?;
```

One HTTPS fetch → fully populated marketplace. Install a plugin → one
more HTTPS fetch for the tarball, no GitHub API calls, no rate limits.

## Update cadence

The `index.json` file is regenerated by the aggregator workflow
in three situations:
- **On `repository_dispatch`** — a plugin repo's release workflow fires
  `aggregate-index` after successfully publishing a GitHub Release.
  This is the primary path and typically completes within 30 s of the
  release going live.
- **On schedule** — every 6 hours as a safety net in case a dispatch
  was missed or rate-limited.
- **On manual `workflow_dispatch`** — for operator-triggered rebuilds.

The JSON is regenerated idempotently; if nothing changed, no commit is
made. If something changed, the file is committed + pushed and
`index.json.sig` is refreshed alongside it.
